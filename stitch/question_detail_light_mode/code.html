Understanding Hidden Classes in V8

alex_dev
Asked 4 hours ago
visibility 1.2k views
42

I'm trying to optimize a high-frequency loop in Node.js and came across the concept of Hidden Classes (or Shapes). I understand that V8 uses them to optimize property access, but I'm struggling with how adding properties in different orders affects performance.

For example, why does this cause a de-optimization? Does V8 create a new class for every property addition?

javascript 
function Point(x, y) {
  this.x = x;
  this.y = y;
}

const p1 = new Point(1, 2);
const p2 = new Point(3, 4);

// This is fine, they share the class
p2.z = 5; 

// Now p1 and p2 have different hidden classes?
p1.z = 6;
v8 compiler optimization

Yes, you're exactly right. V8 creates what is essentially a state machine of classes. Each property addition acts as a transition between these classes.

"Objects that share the same hidden class also share the same optimized code for property access."

In your case, when you add z to p2, V8 transitions it to a new class. When you later add it to p1, it follows the same transition path and ends up on the same hidden class. The problem arises when properties are added in a different order.
